# Prompt Methodology Pivot Plan

## Phase 1

Define the new product spine:

- identify the smallest set of `godex` behaviors that create clear lift
- separate core behavior from old packaging assumptions
- decide the canonical prompt path and naming

## Phase 2

Write the new primary prompt so the target Codex can:

- inspect local durable layers first
- compare local reality against the target behavior set
- skip existing strengths
- add missing durable structures only where needed
- resolve or escalate conflicts conservatively
- verify the resulting local adaptation
- avoid context bloat after installation of the ideas

## Phase 3

Sync product entry surfaces:

- `README.md`
- `README.en.md`
- `prompts/README.md`
- `prompts/bootstrap/README.md`
- `prompts/upgrade/README.md`
- prompt file links and wording

## Phase 4

Run full-text acceptance and repo checks:

- read back the new prompt and entry docs
- verify link targets and wording consistency
- update `CHANGELOG.md`
- review `git diff`
- commit, push, and confirm clean status
