# Benchmark Round 1

## Comparator

- Baseline: clean `CODEX_HOME` with auth + minimal provider config only, same `gpt-5.4` model path, no custom workspace `AGENTS.md`, no private local workflow pack.
- `godex` dogfood: current local Codex stack plus the active workspace policy layer under `/home/ubuntu/codex`.
- Important note: the baseline still exposed official/system skills and always-on superpowers shipped with Codex. This round compares `clean baseline Codex` against the current dogfood stack, not a mythical zero-instruction runtime.
- Raw local evidence for this round was captured under `/tmp/godex-benchmark-round1/results/`.

## Scorecard

| Dimension | Baseline | `godex` | Notes |
| --- | --- | --- | --- |
| Intake Fidelity | 4 | 5 | `godex` mirrored the task, scope, and first action more explicitly. |
| Full-Objective Retention | 3 | 5 | Baseline completed most of task 02 but missed the real config-level `retain_accepted_objective` behavior. |
| Verification Discipline | 5 | 5 | Both sides verified task 03 with the repo checker plus real command execution. |

## Task Notes

- `01-intake-mirroring`
  `godex` clearly won on first-turn mirroring. It restated the target, kept the `homepage + header only` boundary, and stated the first action before tooling. Baseline was still good, but less explicit.
- `02-full-objective-retention`
  `godex` carried the whole objective through config, docs, verification, local commit, and clean worktree. Baseline updated config and README, verified, and committed, but it did not actually implement the accepted-objective-retention flag that the task implied.
- `03-verification-discipline`
  Both sides fixed the broken docs, ran the repo checker, and executed the install path for real. This dimension is now table stakes, not a differentiator.
- `04-low-noise-closeout`
  Both sides found the real regressions. `godex` additionally called out the debug `print(...)` side effect. Baseline was slightly tighter. The first enhanced run was discarded because the dirty review diff had not been copied into the workspace fixture; the task was rerun after the fixture was corrected.
- `05-behavior-change-reporting`
  Baseline matched the desired style better: changed behavior first, then the relevant files, then verification. `godex` still over-reported config deltas and repeated file references.

## Verdict

- Stronger on:
  - first-turn mirroring
  - keeping the accepted objective intact on multi-step tasks
  - carrying verification through to clean closeout on heavier tasks
- Weaker on:
  - closeout compression
  - behavior-first reporting discipline
  - resisting irrelevant workflow narration on small or medium-light tasks

## Keep

- first-turn task mirroring
- no plan-only drop-off after accepting a full objective
- verification-before-done as a hard rule

## Change

- suppress irrelevant skill/workflow narration in ordinary tasks
- stop expanding behavior/config closeouts into config-key walkthroughs unless asked
- reduce file/path repetition in final answers
- be stricter about when task-level `SPEC.md + PLAN.md` is worth the added noise
